## [@anthropic-ai/sdk] @ 0.52.0

- **Use this when**: You need to call Claude models via the Anthropic API for text generation, reasoning, tool use, or message batches—this is the official SDK for Anthropic's hosted models.
- **Prefer over**: @google/genai (Google's models) or openai (OpenAI's models) when the task specifically requires Claude's capabilities, reasoning depth, or tool-use patterns; use this for Anthropic's own API endpoints, not third-party cloud providers.
- **Gotchas**: Version 0.52.0 includes beta APIs (Message Batches, Files) that require explicit `betas` header parameters and may change; streaming with `.stream()` builds a final message object in memory (use `create({ stream: true })` instead for lower memory if you only need events); tool-use responses require you to parse and execute tool calls yourself before sending results back in the next message.
- **NOT for**: Calling Google's Gemini models (use @google/genai), OpenAI's GPT models (use openai), or non-LLM tasks like CLI prompts (use @clack/prompts) or terminal styling (use chalk).

## [@clack/prompts] @ 0.10.1

- **Use this when**: Building interactive command-line interfaces that need text input, yes/no confirmations, single/multi-select menus, spinners, or task runners with a minimal, pre-styled UI.

- **Prefer over**: Raw `readline` or other generic CLI prompt libraries when you want beautiful, opinionated styling out of the box with minimal bundle size. Use instead of building custom prompts with chalk alone — @clack/prompts pairs styling with interactivity, whereas chalk is text color/formatting only.

- **Gotchas**: The `stream` APIs (for dynamic LLM output) expect generators or async iterables; ensure your LLM SDK's streaming responses are properly wrapped or converted to an iterable format. Cancellation detection via `isCancel()` must be checked after every async prompt — forgetting this will cause ungraceful exits when users press Ctrl+C.

- **NOT for**: Programmatic text coloring or formatting independent of prompts (use chalk for that). Server-side or non-terminal environments where user interaction isn't possible. Real-time two-way conversation with users (use a proper chat framework or implement session management separately).

## [dotenv] @ 16.6.1

- **Use this when**: You need to load environment variables from a `.env` file into `process.env` at application startup, typically to manage configuration like API keys, database URLs, or feature flags without hardcoding them.
- **Prefer over**: Manual `process.env` assignment or environment-specific config files — dotenv is the standard for Twelve-Factor App compliance in Node.js projects. Do not use for runtime secrets management; for that, consider external vaults or deployment platform secrets.
- **Gotchas**: `dotenv.config()` must be called as early as possible in your application lifecycle (before other modules that read `process.env`). The `.env` file should not be committed to version control if it contains secrets. Multiline values (like private keys) require proper quoting with `"` delimiters; `\n` escapes work but literal newlines in quotes are safer.
- **NOT for**: Managing secrets across multiple environments or teams in a secure, encrypted manner — use `dotenvx` (a separate tool) for encryption, multi-environment `.env` files, and command substitution. Do not rely on dotenv alone for production deployments requiring secret rotation or audit trails.
